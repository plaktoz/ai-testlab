# GLOSSARY.md Format

A living terminology reference for the teaching workspace. Captures *proven understanding*, not introductory material — a definition earns its place only after the learner demonstrates correct usage.

## Template

```markdown
# Glossary

## <Group (optional, add when natural clusters emerge)>

**<Term>**: <1-2 sentence tight definition stating what the term *is*>.
_Avoid_: <deprecated synonym>, <another alias>

**<Term>**: <definition>.
_Avoid_: <alias>
```

## Rules

- Add a term only when the user understands it — it is a knowledge record, not a resource to learn from.
- Be opinionated: choose one canonical term, relegate synonyms to avoid-lists.
- Use the glossary's own terms inside newer definitions to build compounding clarity.
- Revisit and revise entries as comprehension matures — do not leave stale entries.
- Flag where field usage diverges from workspace usage explicitly.
- Flat lists work until natural clusters emerge, then use subheadings.
