# Advanced LLM / production background

User has built production LLM applications and understands RAG, tool calling, and multi-agent patterns at the usage level. Teaching should go straight to architectural principles and tradeoffs — skip introductory explanations of what an LLM is, how prompting works, or basic API usage.

## Implications for future sessions

- Start lessons at the architectural / design-pattern level
- Use precise terminology without preamble (context window, tool call, orchestrator, etc.)
- Prefer "why does this design choice exist" framing over "here's how to do X"
- Retrieval checks should probe understanding of tradeoffs, not definitions
