# Mission

## Why

Develop a deep, principled understanding of how Agentic AI systems work — architectures, core patterns, design tradeoffs, and failure modes — well beyond surface-level familiarity with tool calling or prompt chaining.

## Success looks like

- Can accurately explain the distinction between workflows and agents, and choose the right pattern for a given problem
- Can describe the main architectural patterns (ReAct, orchestrator-workers, evaluator-optimizer, parallelization) and when each is appropriate
- Can reason about the tradeoffs in memory systems (in-context, external, episodic, semantic)
- Can identify failure modes in agentic systems and design mitigations
- Can evaluate new agent frameworks, papers, or codebases against a coherent mental model
- Can design a multi-agent system at the architecture level, with clear justification for choices

## Constraints

None stated. Theory is in scope where it builds the mental model.

## Out of scope

- Building and shipping production agent systems (this is understand-how-it-works, not build-it)
- Model internals / training (focus is on agent architectures at the system level)
