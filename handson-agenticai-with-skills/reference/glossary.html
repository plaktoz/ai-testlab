Glossary

Terms are added here only after they appear in a lesson. Defined once, used consistently across all lessons.

Core Concepts
Agent Core

A system that perceives its environment, reasons about a goal, and takes autonomous actions — including calling tools and looping — until that goal is met. An agent wraps an LLM in a loop.

Avoid: "AI assistant" (implies passivity), "chatbot" (implies single-turn)
The loop is like a while (!done) driven by a policy, not by you.
Agent Loop Core

The cycle of: (1) send message + tools to the model, (2) execute any tool calls the model requests, (3) return results, (4) repeat — until stop_reason == "end_turn".

Like a while loop with a strategy pattern deciding each iteration.
LLM Core

Large Language Model. A neural network trained on text that generates token-by-token completions. Claude is an LLM. An LLM answers once; an agent loops.

Avoid: "AI" (too broad), "GPT" (a specific competing model)
API & SDK
Message API

A single turn in the conversation. Has a role ("user" or "assistant") and a content (text string or list of content blocks). The messages array is the full conversation history you maintain in your code.

Like a DTO (MessageDTO) passed to and from the API.
System Prompt API

Instructions passed in the system parameter of a message request. Sets the agent's persona, rules, and context. Not part of the conversation history — it is applied globally to every response.

Like a constructor that sets the initial state of your agent object.
stop_reason API

A field on every API response indicating why the model stopped generating. "end_turn" means it finished naturally. "tool_use" means it wants to call one or more tools. "max_tokens" means the output was cut off.

Like a status code on a response object — drives the next branch in your loop.
Tool Use
Tool Tool Use

A Python function that the model can request to have called. Defined as a JSON Schema description; your code executes the actual function. The model never executes code — it only requests execution.

Avoid: "plugin", "function call" (the latter is OpenAI terminology)
Like a @FunctionInterface registered with a dispatcher. The model is the caller; you are the executor.
input_schema Tool Use

A JSON Schema object describing the parameters a tool accepts. Claude reads this to know what arguments to provide when calling the tool. Follows the JSON Schema draft-07 specification.

Like a method signature or a Java record describing the input DTO.
tool_use_id Tool Use

A unique identifier on each tool call block in the assistant's response. Must be echoed back in the tool_result so Claude knows which result belongs to which tool call (important when multiple tools are called in parallel).

Like a correlation ID in a microservices request/response pair.
Tool Result Tool Use

A content block of type "tool_result" sent back to the model after executing a tool. Contains the tool_use_id and the string result of the function call.

Models (current as of 2026)
claude-haiku-4-5-20251001 Model

Anthropic's fastest, most cost-effective model. Ideal for learning, prototyping, high-volume tasks, and anything where speed matters more than maximum reasoning depth. Use this for all lessons unless noted otherwise.

claude-sonnet-4-6 Model

Balanced performance and cost. The go-to model for production applications where quality matters but Opus-level reasoning is not needed.

claude-opus-4-8 Model

Most capable and most expensive. Use for complex reasoning, evaluation, or tasks where correctness matters more than cost.
