# RESOURCES.md Format

A curated, trusted reference set for a given topic. Mirrors the Knowledge/Wisdom split from the teaching philosophy.

## Template

```markdown
# Resources

## Knowledge

- **<Title>** (<Author/Source>) — <what it covers and when to reach for it>
- **<Title>** — <annotation>

## Wisdom (Communities)

- **<Community name>** (<platform>) — <what kind of signal it provides>

## Gaps

- <area where no solid resource has been found yet>
```

## Rules

- High-trust only: primary sources, recognized experts, peer-reviewed work, well-moderated communities. No marketing disguised as education.
- Annotate everything — a bare link is useless in three months.
- Surface gaps with a `## Gaps` section when a needed area has no solid resource.
- Prune actively: remove shallow or wrong sources rather than burying them.
- Log community preferences: if the user opts out of communities, record it to avoid repeat suggestions.
- Better five sharp sources than thirty mediocre ones.
