What is an Agent?

Your first Claude API call in Python. From zero to talking to an AI in one notebook.

⏱ ~20 min 🎯 Simple 🧰 Jupyter Notebook

The big idea

Most software does exactly what you tell it to do — your Java Spring Boot service reads a request, runs your logic, returns a response. It never decides anything.

An AI agent is different. You give it a goal, and it figures out the steps. It can call tools (like a web search or a database), check its own output, and loop until the goal is done — without you scripting every step.

The simplest possible agent has three parts:

  • A model — the brain (Claude, in our case)
  • A loop — runs until the model says it's done
  • Tools — external actions the model can trigger (we add these in Lesson 2)

In this lesson you'll skip the loop and tools — those come later. Right now you'll just talk to the model. One call. One response. That's enough to understand the foundation.

Key insight

An LLM answers a single question. An agent keeps going until a goal is met. Today we build the LLM layer. Tomorrow we wrap it in the loop.

Java Developer Bridge

Think of the Claude API like calling a remote service — similar to how you'd call a REST endpoint with RestTemplate or WebClient in Spring Boot. The Python anthropic package is the SDK that wraps those HTTP calls for you, just like the Feign client wraps REST calls in Java.

Anthropic also ships an official Java SDK (com.anthropic:anthropic-java) which has the same structure. Once you understand the Python version, the Java version will feel immediately familiar.

Step-by-step

  1. Get your API key

    Go to console.anthropic.com, sign in, and create an API key. Copy it somewhere safe — you'll only see it once.

    Security

    Never commit your API key to Git. Store it as an environment variable, not in your notebook code.

  2. Set your API key as an environment variable

    Open your terminal and run:

    Terminal
    # macOS / Linux
export ANTHROPIC_API_KEY="sk-ant-…your-key…"

# Windows (Command Prompt)
set ANTHROPIC_API_KEY=sk-ant-…your-key…

# Or add to your shell profile (~/.zshrc or ~/.bashrc) so it persists

    The anthropic Python SDK automatically reads this variable — you don't pass the key in your code.

  3. Install the SDK and launch Jupyter
    Terminal
    # Install dependencies (once)
pip install anthropic jupyter

# Launch Jupyter in this folder
jupyter notebook

    Jupyter will open in your browser. Click New → Python 3 (ipykernel) to create a fresh notebook. Name it lesson-01-hello-agent.ipynb.

  4. Import the SDK

    In the first cell of your notebook, type and run (Shift+Enter):

    Cell 1 — Code
    import anthropic

# Create a client — it reads ANTHROPIC_API_KEY from the environment
client = anthropic.Anthropic()

    No output means success. If you see ModuleNotFoundError, run pip install anthropic again in the terminal where Jupyter is running.

  5. Make your first API call

    Add a new cell and run this:

    Cell 2 — Code
    response = client.messages.create(
    model="claude-haiku-4-5-20251001",  # fast and cheap — great for learning
    max_tokens=1024,
    messages=[
        {"role": "user", "content": "What is an AI agent? Explain in 2 sentences."}
    ]
)

print(response.content[0].text)
    Expected Output (will vary)
    An AI agent is a system that perceives its environment and takes
autonomous actions to achieve specific goals. Unlike simple programs
that follow fixed rules, agents can adapt their behavior based on
context and feedback to accomplish complex tasks.
    What's happening

    client.messages.create() sends your message to Claude and waits for a reply. response.content[0].text extracts the text from the first content block. The response can have multiple blocks (we'll see that in Lesson 2 with tools).

  6. Inspect the full response object

    Don't just trust the text. Look at what the API actually returns:

    Cell 3 — Code
    # Print the full response to understand the structure
print("Stop reason:", response.stop_reason)
print("Model:",       response.model)
print("Input tokens:", response.usage.input_tokens)
print("Output tokens:",response.usage.output_tokens)
    Expected Output
    Stop reason: end_turn
Model: claude-haiku-4-5-20251001
Input tokens: 22
Output tokens: 47

    stop_reason: "end_turn" means the model finished naturally. You'll later see "tool_use" when the model wants to call a tool — that's the signal to keep the loop going.

  7. Try a system prompt

    A system prompt gives the agent its persona and instructions. Think of it as the constructor of a Java object — it sets the starting state.

    Cell 4 — Code
    response = client.messages.create(
    model="claude-haiku-4-5-20251001",
    max_tokens=512,
    system="You are a concise assistant who only answers in bullet points.",
    messages=[
        {"role": "user", "content": "What are the 3 parts of an AI agent?"}
    ]
)

print(response.content[0].text)
    System prompt vs user message

    The system parameter sets the agent's role and rules — it is not part of the conversation history. The messages list is the conversation. Every message has a role ("user" or "assistant").

Which model should I use?

Anthropic offers three tiers. During learning, always start with the cheapest:

Model ID When to use
Haiku 4.5 claude-haiku-4-5-20251001 Learning, prototyping, fast/cheap tasks
Sonnet 4.6 claude-sonnet-4-6 Balanced quality/cost for real apps
Opus 4.8 claude-opus-4-8 Complex reasoning, most capable tasks
Java Developer Bridge — Side-by-side

Here is the same call in Java using the official Anthropic Java SDK:

Java (Anthropic SDK)
// Maven: com.anthropic:anthropic-java:<latest>
import com.anthropic.client.AnthropicClient;
import com.anthropic.client.okhttp.AnthropicOkHttpClient;
import com.anthropic.models.messages.*;

AnthropicClient client = AnthropicOkHttpClient.builder()
    .apiKey(System.getenv("ANTHROPIC_API_KEY"))
    .build();

var response = client.messages().create(
    MessageCreateParams.builder()
        .model(Model.CLAUDE_HAIKU_4_5_20251001)
        .maxTokens(1024)
        .addUserMessage("What is an AI agent? Explain in 2 sentences.")
        .build()
);

System.out.println(response.content().get(0).text().orElse(""));

The structure is identical — same concepts, same parameter names, just Java syntax. The stop_reason, usage, and model fields are all on the response object in both SDKs.
