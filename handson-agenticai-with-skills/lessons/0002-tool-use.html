Tool Use

Give your agent the ability to call Python functions. This is what makes it truly agentic.

⏱ ~25 min 🎯 Simple 🧰 Jupyter Notebook

The big idea

In Lesson 1, Claude answered questions from its training data alone. But a real agent needs to act — check today's date, call an API, read a file, query a database. That's what tools are for.

You define a tool as a Python function + a JSON description of its parameters. Claude reads the description, decides when to call the tool, and tells you what arguments to pass. Your code calls the actual function and sends the result back. Claude then continues reasoning with the new information.

The key mental model

Claude never calls your function directly — you call it. Claude says "please call get_weather(city='Singapore')", and your agent loop executes that, returns the result, and Claude continues. Claude is the brain; your code is the hands.

The Agent Loop with tools

┌─────────────────────────────────────────────┐
│  1. Send message (+ available tools)        │
│  2. Claude responds                         │
│     ├── stop_reason = "end_turn"  ──────────┼── Done. Print result.
│     └── stop_reason = "tool_use" ──────────┐│
│  3. Execute the requested tool(s)           ││
│  4. Send tool result back to Claude    ◄────┘│
│  5. Go to step 2                            │
└─────────────────────────────────────────────┘
Java Developer Bridge

Tool use is like callback interfaces in Java. You define an interface (the tool schema), Claude decides to call it, your code implements it, and the result flows back. It is also similar to how Java CDI/dependency injection lets a framework call your @Bean methods — you define the contract, the framework decides when to invoke.

Step-by-step

Continue in the same notebook (lesson-01-hello-agent.ipynb) or create lesson-02-tool-use.ipynb.

  1. Define a tool function

    Write a Python function you want the agent to be able to call:

    Cell 1 — Code
    import anthropic
import datetime

client = anthropic.Anthropic()

# The actual Python function
def get_current_date():
    """Return today's date as a string."""
    return datetime.date.today().isoformat()  # e.g. "2026-06-21"

# Tell Claude about the tool using a JSON schema
tools = [
    {
        "name": "get_current_date",
        "description": "Returns today's date in ISO format (YYYY-MM-DD).",
        "input_schema": {
            "type": "object",
            "properties": {},  # no parameters needed
            "required": []
        }
    }
]

    The input_schema is a JSON Schema object. Claude reads it to know what arguments to provide when calling the tool. For get_current_date, no arguments are needed.

  2. Write the agent loop
    Cell 2 — Code
    def run_agent(user_message):
    messages = [{"role": "user", "content": user_message}]

    while True:
        response = client.messages.create(
            model="claude-haiku-4-5-20251001",
            max_tokens=1024,
            tools=tools,
            messages=messages
        )

        # Finished naturally — return the text
        if response.stop_reason == "end_turn":
            return response.content[0].text

        # Claude wants to call a tool
        if response.stop_reason == "tool_use":
            # Add Claude's response to the message history
            messages.append({"role": "assistant", "content": response.content})

            # Execute each tool Claude requested
            tool_results = []
            for block in response.content:
                if block.type == "tool_use":
                    print(f"→ Calling tool: {block.name}({block.input})")

                    # Dispatch to the right function
                    if block.name == "get_current_date":
                        result = get_current_date()

                    tool_results.append({
                        "type": "tool_result",
                        "tool_use_id": block.id,
                        "content": str(result)
                    })

            # Send tool results back to Claude
            messages.append({"role": "user", "content": tool_results})
  3. Run your agent
    Cell 3 — Code
    answer = run_agent("What day of the week is today? Use the tool to get the date.")
print(answer)
    Expected Output
    → Calling tool: get_current_date({})
Today is Saturday, June 21, 2026.

    Notice the loop ran twice: once to get Claude to decide to use the tool, and once after returning the result so Claude could write the final answer.

  4. Add a tool with parameters

    Now define a tool that takes an argument — a simple calculator:

    Cell 4 — Code
    def calculate(expression: str) -> str:
    """Safely evaluate a simple arithmetic expression."""
    try:
        # eval() is safe here because we restrict it to numbers/operators
        allowed = set("0123456789+-*/().,% ")
        if not all(c in allowed for c in expression):
            return "Error: only arithmetic allowed"
        return str(eval(expression))
    except Exception as e:
        return f"Error: {e}"

calc_tool = {
    "name": "calculate",
    "description": "Evaluate a basic arithmetic expression and return the result.",
    "input_schema": {
        "type": "object",
        "properties": {
            "expression": {
                "type": "string",
                "description": "The arithmetic expression to evaluate, e.g. '42 * 1.15'"
            }
        },
        "required": ["expression"]
    }
}
  5. Wire multiple tools and test
    Cell 5 — Code
    def dispatch_tool(name, inputs):
    """Map tool name → function call."""
    if name == "get_current_date": return get_current_date()
    if name == "calculate":       return calculate(inputs["expression"])
    return f"Unknown tool: {name}"

def run_agent_v2(user_message, all_tools):
    messages = [{"role": "user", "content": user_message}]
    while True:
        response = client.messages.create(
            model="claude-haiku-4-5-20251001",
            max_tokens=1024,
            tools=all_tools,
            messages=messages
        )
        if response.stop_reason == "end_turn":
            return response.content[0].text
        messages.append({"role": "assistant", "content": response.content})
        tool_results = []
        for block in response.content:
            if block.type == "tool_use":
                result = dispatch_tool(block.name, block.input)
                print(f"  Tool: {block.name}({block.input}) → {result}")
                tool_results.append({"type": "tool_result",
                                      "tool_use_id": block.id,
                                      "content": str(result)})
        messages.append({"role": "user", "content": tool_results})

# Test it
print(run_agent_v2(
    "What is 15% tip on a $48.50 dinner bill? Also tell me today's date.",
    [tools[0], calc_tool]
))
    Expected Output
      Tool: calculate({'expression': '48.50 * 0.15'}) → 7.275
  Tool: get_current_date({}) → 2026-06-21
A 15% tip on a $48.50 dinner bill is $7.28.
Today's date is June 21, 2026.

Java Developer Bridge — Tool Dispatch Pattern

The dispatch pattern above (name → function call) is equivalent to a strategy pattern in Java:

Java equivalent
// Java tool dispatcher using a Map of strategies
Map<String, Function<Map<String, Object>, String>> tools = new HashMap<>();
tools.put("calculate", inputs -> calculate((String) inputs.get("expression")));
tools.put("get_current_date", inputs -> getCurrentDate());

// Dispatch
String result = tools.get(toolName).apply(toolInputs);

The JSON input_schema maps to a Java record or DTO class. In the Anthropic Java SDK, tool results are typed objects rather than raw strings.
